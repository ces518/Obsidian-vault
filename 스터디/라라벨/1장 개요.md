## 1. 라라벨이란?

- **PHP 기반 웹 애플리케이션 프레임워크**
- 개발자: **Taylor Otwell**
- MVC 아키텍처 기반
- “**개발 생산성과 가독성**”을 최우선 가치로 설계됨

---

## 1-1. 라라벨의 특징

### 1) 쉬운 학습 곡선

- **Facade 패턴**을 적극 사용
    
    → 정적 메서드 호출처럼 보이지만 내부적으로는 DI 컨테이너를 통해 실제 인스턴스 메서드 실행
    
- 프레임워크 내부 복잡성은 숨기고, **사용자 API는 단순하게 제공**
    

```php
$name =\\Session::get("name");
```

> 사용자는 구현 디테일을 몰라도 기능을 빠르게 사용 가능

---

### 2) Symfony 기반

- 라라벨 코어는 **Symfony 컴포넌트 위에서 동작**
- 검증된 컴포넌트를 사용 → 안정성과 확장성 확보

---

### 3) 다양한 기본 기능 제공

- 라우팅, 컨트롤러, 뷰, ORM(Eloquent)
- 인증 / OAuth
- 이벤트 / 큐
- 테스트 (PHPUnit, Dusk)
- DI 컨테이너, 서비스 프로바이더

👉 **“웹 서비스에 필요한 건 거의 다 있다”**

---

### 4) 활발한 버전 업그레이드

- 과거: **6개월 주기 메이저 릴리스**
- Laravel 9 이후: **1년 주기**
- **LTS**
    - 2년마다 출시
    - 버그 픽스 2년 + 보안 패치 3년

> 실무 프로젝트에 장기 도입 가능

---

### 5) 높은 확장성

- Composer 기반 패키지 관리
- PSR 표준 & 자동 로딩
- 디렉터리 구조를 강제하지 않음
- 팀/조직 규모에 맞게 아키텍처 확장 가능

## 2. 환경 구축

### 1) 기본 개념

- 로컬 PC(macOS / Windows)에 **가상 환경**
- 그 위에서 Laravel 실행

---

### 2) Laravel Sail

- **Docker 기반 공식 개발 환경**
- 포함 요소:
    - PHP
    - MySQL
    - Redis
    - MailHog
    - Selenium
    - Node
- `sail` 명령어 = 내부적으로 docker-compose 호출

주요 명령:

```bash
sail up -d
sail down
sail shell
sail ps
sail mysql
```

👉 **가볍고 표준적인 개발 환경**

---

### 3) Homestead

- Vagrant + VirtualBox 기반
- 더 많은 구성요소 포함:
    - nginx
    - PostgreSQL / SQLite
    - Memcached
    - Gulp 등
- 비교적 **무겁고 설정 복잡**

👉 최근 실무에선 **Sail 선호**

## 3. 첫 번째 애플리케이션

### 3-1. 라라벨 디렉터리 구조

핵심 디렉터리 역할:

|디렉터리|역할|
|---|---|
|app|컨트롤러, 미들웨어, 서비스 로직|
|routes|라우팅 정의|
|resources|뷰(Blade), 프론트 리소스|
|public|문서 루트, entry point|
|database|마이그레이션, 시더|
|tests|테스트 코드|
|vendor|Composer 패키지|

---

### 3-2. Welcome 페이지 처리 흐름

**요청 흐름**

1. `public/index.php`
2. 라우팅 (`routes/web.php`)
3. 컨트롤러 or 클로저
4. 뷰 반환 (`welcome.blade.php`)

```php
Route::get('/', function () {returnview('welcome');
});
```

---

### 3-3. 첫 페이지 구현

- `/home` 라우팅 추가
- `home.blade.php` 생성
- 기본 HTML + Blade 템플릿 사용

---

### 3-4. 테스트 코드

- 기본 테스트 프레임워크: **PHPUnit**
- Feature Test 작성

```php
$response =$this->get('/home');$response->assertStatus(200);$response->assertSeeText("안녕하세요!");
```

- 실행:

```bash
sailtest
```

👉 **HTTP 레벨 통합 테스트가 기본 문화**
